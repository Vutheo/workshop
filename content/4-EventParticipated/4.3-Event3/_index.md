# Event Report: "FCAJ Community Day - June 2026"

### Event Objectives

- Update participants on the latest trends in AI and Cloud Computing for enterprise applications.
- Share career development, startup experiences, and practical AI product development.
- Introduce AI Agent, Voice AI, and DevOps Agent solutions built on AWS.
- Help students understand modern AI deployment models and the requirements for deploying AI systems in production environments.

## Key Highlights

### Career Development and Startup Journey

The speaker shared his journey from a system operations engineer to becoming an AWS Solution Architect, as well as the story behind founding CloudThinker.

- Shared experiences in learning AWS, earning AWS certifications, and following the cloud transformation trend.
- Explained how AI is reshaping the job market, where companies increasingly seek highly skilled engineers who can effectively leverage AI.
- Encouraged students to participate in internships early to gain practical experience and improve their competitiveness.
- Introduced the CloudThinker platform, which provides AI-powered Incident Management, FinOps, Security Assessment, and infrastructure analysis.
- Explained the advantages of adopting a **Multi-Agent System** to divide responsibilities, optimize context, reduce costs, and improve access control.
- Emphasized that successful startups should begin with real business problems and quickly transform ideas into practical solutions.

### Building Scalable Voice Agents

This session focused on designing Voice AI systems on AWS and the challenges of supporting the Vietnamese language.

- Introduced two common architectures: **Speech-to-Speech** and **Speech-to-Text → LLM → Text-to-Speech**.
- Explained why Vietnamese enterprises generally prefer the Speech-to-Text architecture combined with LLMs.
- Demonstrated how to build a Voice Agent using **AWS Bedrock AgentCore** and **Knowledge Base**.
- Discussed technical requirements such as real-time streaming, speaker gender recognition, interruption handling, Tool Calling, and prompt management.
- Highlighted that Voice Agents should collaborate with human operators and seamlessly transfer conversations when necessary.

### DevOps Agent – An AI Assistant for System Operations

The speakers introduced **AWS DevOps Agent**, an AI solution that assists DevOps engineers in monitoring systems and investigating incidents.

- Learns system architecture through Topology, Context, Memory, and Skills.
- Integrates with AWS services to collect logs, metrics, and traces for troubleshooting.
- Supports the workflow of incident detection, root cause analysis, solution recommendations, and system improvement.
- Demonstrated how the agent investigated a DDoS attack and generated a remediation plan within a short time.
- Explained that the DevOps Agent helps reduce MTTR and MTTD while still requiring human approval before executing actions.

### AI Applications in Enterprise Human Resource Management

This session introduced how **Amazon Q** can improve recruitment and HR management.

- Automatically screens resumes, matches candidates with job descriptions, and evaluates candidate suitability.
- Generates reports and supports data-driven recruitment workflows.
- Connects with multiple data sources to automate repetitive tasks.
- Helps HR teams improve recruitment efficiency while saving time.

### Building a Secure Private MCP for Amazon Q

The final session presented a secure **Private MCP Server** architecture that enables Amazon Q to access internal enterprise data safely.

- Introduced an architecture based on Amazon VPC, Private DNS, ALB, and VPC Connection.
- Ensures that sensitive enterprise data remains within private networks instead of being transmitted over the public Internet.
- Demonstrated Amazon Q securely accessing internal enterprise resources through the MCP Server.
- Shared deployment costs and practical considerations for enterprise environments.
